─── 3.10 WEBSOCKET SERVER (main handler) ───────────────────────────────────
async def handle_client(websocket, path=None):
    """Main WebSocket connection handler."""
    
    On connection:
    1. Send {"type": "event", "event": "connection_ready", "data": {"server_version": "1.0.0", "available_features": [...]}, "timestamp": "..."}
    2. Start system status push loop (every 5 seconds):
       → {"type": "event", "event": "system_status", "data": {"cpu": 45.2, "ram": 62.1, "battery": {"percent": 78, "plugged": true}}, "timestamp": "..."}
    3. Start clipboard monitor loop (every 2 seconds, check if clipboard changed)
    4. Start battery alert monitor (alert when < 20%)

    On message received:
    1. Parse JSON, validate structure
    2. Log command to database
    3. Check rate limit
    4. Route to appropriate handler based on category+action
    5. If action requires confirmation AND no confirmation_id in params:
       → Return confirmation_required response
    6. If it's a confirmation approval:
       → Look up pending confirmation, execute original command
    7. If it's a confirmation rejection:
       → Delete pending confirmation, return cancelled message
    8. For "voice_start":
       → Start VoiceEngine listening, send partial/final results as events
    9. For "voice_stop":
       → Stop listening
    10. For "tts_speak":
       → Use VoiceEngine to generate audio, send as base64 chunks
    11. For "query" type (text chat):
       → Check command patterns first (NLU)
       → If pattern matches → execute PC command
       → If no pattern matches → send to LLMRouter for conversation
       → Stream LLM response back as llm_stream messages
    12. For "config" type:
       → Update runtime config (switch provider, change voice settings, etc.)
    13. For "cancel":
       → Cancel any ongoing operation (TTS, LLM stream, etc.)

    On disconnection:
    1. Stop all loops for this client
    2. Clean up resources

async def main():
    """Start WebSocket server."""
    server = await websockets.serve(handle_client, "0.0.0.0", 8765)
    print("🔗 Pika AI Bridge running on ws://localhost:8765")
    await server.wait_closed()

if __name__ == "__main__":
    asyncio.run(main())

─── 4.5 useWebSocket HOOK ──────────────────────────────────────────────────
- Connect to WebSocket URL from store.settings.bridgeUrl (default: ws://localhost:8765)
- Auto-reconnect with exponential backoff (1s, 2s, 4s, 8s, max 30s)
- On message: parse JSON, dispatch to store based on type:
  - "response" → addMessage or updateLastMessage, handle errors, handle confirmation_required
  - "event" → handle each event type:
    - "connection_ready" → setConnected('connected')
    - "system_status" → setSystemStatus
    - "voice_partial" → setPartialTranscript
    - "voice_final" → setFinalTranscript, process as command
    - "tts_started" → setSpeaking(true)
    - "tts_ended" → setSpeaking(false)
    - "tts_audio_chunk" → play audio chunk (base64 decode → AudioContext)
    - "reminder_triggered" → addToast + notification sound
    - "battery_alert" → addToast warning
    - "clipboard_changed" → (optional) show small notification
  - "llm_stream" → if !done: updateLastAssistantMessage(chunk); if done: finalize message
- Expose: send(message), connect(), disconnect(), isConnected
- Send heartbeat every 30 seconds: {"type": "ping"}

─── 4.6 useVoice HOOK ──────────────────────────────────────────────────────
- Uses Web Audio API + getUserMedia to capture microphone audio
- DOES NOT do STT locally — sends "voice_start"/"voice_stop" to backend
- Backend does Vosk STT and sends back partial/final events
- Generates fake waveform data for visualization (random bars when listening)
- Expose: startListening(), stopListening(), isListening

─── 4.7 useSpeech HOOK ─────────────────────────────────────────────────────
- Uses Web SpeechSynthesis API as fallback TTS (when backend TTS not available)
- Expose: speak(text, lang), stop(), isSpeaking
- Also handles backend TTS audio chunk playback via AudioContext
