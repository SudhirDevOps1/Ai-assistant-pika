1. Requirements — Kya Chahiye

Software Requirements

  • Node.js v18+ — JavaScript runtime (check: node -v)

  • npm — Package manager (Node ke saath aata hai)

6b. Pika AI Dashboard & Project Architecture Guide

Pika AI 3-Column Grid Interface Layout

Naya futuristic matte-black dashboard 3 columns mein structured hai:

  • Left Column (Diagnostics & Stack): Network Telemetry, Core Metrics (CPU, RAM, Temp, OS), Webcam Feed (purple toggle), Built With the Best tech specs.

6d. WebSocket API Protocol & Command JSON Reference

Bidirectional JSON Messaging Schema

Next.js Web UI aur Python Bridge ke beech saari communication standardized JSON payloads ke zariye hoti hai. Jab aap UI button dabate hain ya voice command bolte hain, client ek structured payload bhejta hai, aur server turant action execute karke status aur telemetry response return karta hai.

1. Volume & System Control Command (Client ➔ Server)

JSON Request Schema
{ "action": "system_control", "command": "set_volume", "value": 75, "timestamp": 1712400000000 }

2. App Execution & URL Launch Command (Client ➔ Server)

JSON Request Schema
{ "action": "app_launcher", "app_name": "chrome", "url": "https://github.com/SudhirDevOps1/Ai-assistant-pika" }

3. Real-Time Telemetry Stream Response (Server ➔ Client)

JSON Response Schema
{ "status": "success", "type": "telemetry", "data": { "cpu_percent": 18.4, "ram_gb": { "used": 7.8, "total": 16.0, "percent": 48.7 }, "battery": { "percent": 92, "charging": true }, "os": "Windows 11 Pro (64-bit)" } }

6e. 4-Tier Voice & AI Execution Pipeline Mechanics

How Hindi/English Voice Commands Are Processed

Jab aap bole hain "Hey Pika, Chrome kholo aur gaana bajao", tab system ek ultra-low latency 4-tier processing architecture execute karta hai:

1. Mic Input
Web Speech API / Vosk
➔
2. AI Intent Parse
Groq Llama 3.3 70B
➔
3. Action Execute
WebSocket Bridge
➔
4. Voice Feedback
Edge-TTS / Pyttsx3
  • Tier 1 — Speech-To-Text (STT): Browser microphone stream ko native webkitSpeechRecognition API ya offline Python vosk model se capture karta hai. Yeh automatically Hinglish, Hindi aur English ke accents ko accurately transcribe karta hai.
  • Tier 2 — AI Prompt Engineering & Intent Extraction: Transcribed text ko Groq API (ya Gemini/Mistral) pe system prompt ke saath bheja jata hai. System prompt AI ko instruct karta hai ki user ka intent samjhe aur agar koi system action chahiye (jaise volume ya app launch), toh text ke saath JSON tool payload embed kare.
  • Tier 3 — Zero-Latency Hardware Execution: Web UI AI response se JSON tool payload extract karke Python Bridge ko WebSocket pe bhej deta hai jo turant system hardware ko command bhejta hai.
  • Tier 4 — Smart Hybrid Text-To-Speech (TTS): Bolne ke liye system pehle ek 0.6s DNS query se check karta hai ki internet connected hai ya nahi. Agar high-speed internet hai, toh Microsoft Edge-TTS neural voice (hi-IN-SwaraNeural ya en-US-Neural2-A) se natural reply sunata hai. Agar offline ya network slow hai, toh bina kisi delay ke local OS voice engine (pyttsx3) pe switch ho jata hai!

11. Troubleshooting — Fix Problems

"No API key configured"

Fix: Settings → API key daalo → Save Settings. Key console.groq.com se milegi.

PC Control kaam nahi kar raha

  • Python bridge chal raha hai? (python pc_bridge.py)
  • Settings mein ws://localhost:8765 daala?
  • Save Settings button dabaya?
  • Firewall allow kiya? (Windows Defender → Allow an app)

npm run dev error aa raha

Fix Commands
rmdir /s /q node_modules del package-lock.json npm install npm run dev

Port 3000 already in use

Kill Port
npx kill-port 3000 npm run dev

Python bridge Unicode error

Terminal
chcp 65001 python pc_bridge.py

Volume/Media controls kaam nahi

Fix: pip install pyautogui — yeh package zaroori hai volume, media, aur keyboard controls ke liye.

Voice recognition kaam nahi kar raha

Fix:

  • Browser ko microphone permission do (lock icon → Allow)
  • Chrome ya Edge use karo (Firefox mein limited support)
  • HTTPS ya localhost pe hona chahiye
